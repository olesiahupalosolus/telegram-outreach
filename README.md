# Telegram Outreach

Багатокористувацький веб-інструмент для масової розсилки персоналізованих повідомлень у Telegram через ваші особисті акаунти. Кожен користувач має власний обліковий запис, свої Telegram-акаунти і свою чергу розсилок — дані ізольовані. Контакти і тексти повідомлень підтягуються з Google Sheets.

## ⚠️ Важливо перед використанням

1. **Telegram ToS:** масова розсилка незнайомим людям порушує [Terms of Service Telegram](https://telegram.org/tos) і може призвести до бану акаунта.
2. **GDPR / закон про персональні дані:** холодний DM комерційного характеру до людей, які не давали згоди, регулюється законом. Перевірте дозволи у вашій юрисдикції.
3. **Безпека секретів:** ніколи не комітьте справжні `api_id`, `api_hash`, файл бази даних `app.db` або файл `.env` з ключами.

## Стек

- Python 3 + Flask
- Flask-Login (авторизація) + Flask-SQLAlchemy (ORM)
- PostgreSQL (продакшн на Railway) / SQLite (локально як fallback)
- Telethon з `StringSession` (сесії зберігаються в БД, переживають рестарт)
- pandas (читання Google Sheets через CSV-експорт)
- Деплой: Railway (gunicorn + Postgres plugin)

## Як працює багатокористувацька модель

- **Реєстрація — invite-only.** Перший користувач, що зареєструвався, автоматично стає адміністратором (bootstrap). Усі наступні мають ввести код запрошення.
- **Адмін** має сторінку `/admin/invites`, де може створювати одноразові коди й ділитися посиланням `…/register?code=XXXX`.
- **Кожен юзер** бачить тільки СВОЇ Telegram-акаунти, СВОЇ розсилки і СВІЙ статус. Дані інших користувачів недоступні.

## Локальний запуск

```bash
# 1. Встановити залежності
pip install -r requirements.txt

# 2. Створити .env з власним секретом
cp .env.example .env
python -c "import secrets; print('SECRET_KEY=' + secrets.token_hex(32))" >> .env

# 3. Запустити
python app.py
```

Відкрити http://localhost:8080 — система автоматично створить SQLite-файл `app.db` і пустить вас на сторінку реєстрації. Перший зареєстрований стане адміном.

## Як користуватись

1. **Зареєструватися** — перший раз без коду (bootstrap), далі — за інвайтом.
2. **Додати Telegram-акаунт** — натисніть "+ Додати акаунт" і введіть `api_id`, `api_hash` з [my.telegram.org](https://my.telegram.org), номер телефону, код підтвердження (і пароль 2FA, якщо ввімкнено). Сесія збережеться у вашому профілі.
3. **Підготувати Google Sheet** — таблиця з двома колонками: `@username` отримувача і текст повідомлення. Відкрийте доступ "Переглядач за посиланням".
4. **Запустити розсилку** — вставте посилання на таблицю, виберіть колонки, акаунт і затримку (рекомендовано ≥20 сек), натисніть "Запустити розсилку".

## Деплой на Railway

1. **Підключити Postgres plugin:**
   - У вашому Railway-проєкті: `+ New` → `Database` → `Add PostgreSQL`.
   - Railway автоматично створить змінну оточення `DATABASE_URL` у вашому web-сервісі.
2. **Встановити змінні оточення в web-сервісі:**
   - `SECRET_KEY` — згенерований випадковий рядок (`python -c "import secrets; print(secrets.token_hex(32))"`).
   - `DATABASE_URL` — підтягнеться автоматично з Postgres-сервісу (`${{Postgres.DATABASE_URL}}`), нічого вручну вписувати не треба.
3. **Перший заход:** після деплою відкрийте URL → `/register` → зареєструйте перший акаунт (стане адміном).
4. **Запросити інших:** з облікового запису адміна перейдіть до `🎟 Інвайти`, згенеруйте код, скопіюйте `…/register?code=XXXX` посилання й надішліть людині.

⚠️ Telethon `StringSession` зберігається в Postgres, тому переживає рестарти і деплої. Без Postgres (наприклад, на ефемерному файловому SQLite на Railway) сесії губляться при кожному redeploy.

## Структура

| Файл | Опис |
|---|---|
| `app.py` | Flask-додаток: моделі (User / InviteCode / TelegramAccount), Flask-Login, Telethon-логіка |
| `templates/index.html` | Головна — список акаунтів, превʼю Sheet, запуск розсилки |
| `templates/login.html` / `register.html` | Сторінки авторизації |
| `templates/admin_invites.html` | Адмін-панель інвайт-кодів |
| `app.db` | Локальний SQLite-файл (gitignored) |
| `.env.example` | Шаблон змінних оточення |
| `requirements.txt` | Python-залежності |
| `Procfile`, `railway.json` | Конфіг деплою на Railway |
